**1. Schemas (`packages/supabase/src/schemas/group.ts`)** — mirror `packages/supabase/src/schemas/country.ts` shape. Define and export:

**2. DAL (`packages/supabase/src/dal/group.ts`)** — `import 'server-only'` at top. Use `createClient` (cookie-authed) — never service-role on reads (RESEARCH.md don't-hand-roll table). Pull row types from `Database['public']['Views'][...]['Row']` and `Database['public']['Functions']['group_speed_to_lead_series']['Returns'][number]` so the regenerated 00013 types are the source of truth. Export:

- `getGroupTodayStats(): Promise<GroupTodayStats>` — `.from('group_today_stats').select('*').single()`. Throws on error.
- `getCountryPerformanceToday(): Promise<CountryPerformanceRow[]>` — `.from('country_performance_today').select('*')`. Already ordered by `total_leads DESC` at the view layer; preserve order in the array (supabase-js does).
- `getLeadsByServiceGroup(): Promise<LeadsByServiceGroupRow[]>` — `.from('leads_by_service_group').select('*')`. Ordered `leads_count DESC` at view layer.
- `getGroupSpeedToLeadSeries(p_days: number = 7): Promise<GroupSpeedToLeadDay[]>` — `.rpc('group_speed_to_lead_series', { p_days })`. Validates `p_days` is a positive int via Zod before calling. Throws on error.
- `computeResponseStatus(seconds: number | null): 'green' | 'amber' | 'red'` — pure function, single source of truth for the status-dot bucket. Per RESEARCH.md open question 2 (recommendation: fixed in DAL): `null` → red; `< 5 min` → green; `≤ 8 min` → amber; `> 8 min` → red. Constants exported as `RESPONSE_STATUS_THRESHOLDS = { green: 300, amber: 480 }` (seconds) so the legend in the leaderboard can import the same numbers without hardcoding.

**3. Barrel update (`packages/supabase/src/dal/index.ts`)** — re-export the new symbols and types alongside the existing country/queue exports.

**4. Tests (`apps/web/tests/hq.dal.test.ts`)** — 5–6 vitest cases:

   - `getCountryPerformanceToday()` returns 12 rows for HQ; rows are ordered by total_leads desc
   - `getLeadsByServiceGroup()` total reconciles to `total_leads_group` from the today-stats view (sanity)
   - `getGroupSpeedToLeadSeries(7)` returns 0..7 rows for HQ; throws/rejects for country_admin (RPC role guard from 05-01)
   - `computeResponseStatus()` — table-driven cases: `null` → red, `0` → green, `299` → green, `300` → amber, `480` → amber, `481` → red, `1000` → red
   - Optional: `computeResponseStatus(5 * 60 - 1)` and `computeResponseStatus(5 * 60)` to pin the boundary
