    1. **`apps/web/app/api/leads/ingest/route.ts`** â€” implement the webhook handler exactly per RESEARCH Pattern 4 ("Webhook handler order of operations"):
       - `export async function POST(req: Request)`
       - Read body once: `const raw = await req.text()`
       - Compute HMAC-SHA256 hex digest of raw using `process.env.PARATUS_INGEST_SECRET` (throw 500 if env missing)
       - Read `x-paratus-signature` header; convert both digests to equal-length `Buffer`s; compare with `crypto.timingSafeEqual`
       - On signature mismatch: return `Response('invalid signature', { status: 401 })`
       - Parse JSON: `JSON.parse(raw)` inside a try/catch; on parse error 400
       - Validate with `ingestSchema.safeParse(parsed)`; on failure return `Response.json({ error: result.error.flatten() }, { status: 400 })`
       - Call `ingestLead(result.data)`; on returned error object return 422; on success return 201 with `{ lead_id, agent_id, duplicate }` (200 if `duplicate=true`).
       - Wrap the entire handler in a try/catch logging structured `{ event: 'ingest_error', message, stack }` to stderr; return generic 500 to caller.
       - Add `export const runtime = 'nodejs'` to ensure Node runtime (HMAC needs it).
       - Do NOT use `dynamic = 'force-dynamic'` â€” Route Handlers are dynamic by default.
