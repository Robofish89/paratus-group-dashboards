  <action>
    1. **`apps/web/tests/ingest.idempotency.test.ts`** — node test that:
       - Computes HMAC over a fixed body with `process.env.PARATUS_INGEST_SECRET`.
       - POSTs to `http://localhost:3012/api/leads/ingest` (or `process.env.INGEST_TEST_URL` to allow staging).
       - Asserts first response is 201 with `duplicate:false`.
       - POSTs identical body again. Asserts 200 with `duplicate:true` and SAME `lead_id`.
       - POSTs with one byte tweaked in signature. Asserts 401.
       - POSTs with malformed body. Asserts 400.

    2. **`apps/web/tests/realtime.broadcast.test.ts`** — node test that:
       - Signs in browser client as the agent test user.
       - Subscribes to channel `agent:<agent_uid>` with `{ config: { private: true } }`.
       - Awaits a `subscribed` state.
       - Triggers an ingest (POST a fresh lead via the route handler with valid HMAC, country MZ — round-robin should pick the agent since they're the only MZ agent).
       - Asserts a broadcast event arrives within 5s with `payload.record.country_code === 'MZ'`.
       - Cleans up the channel.

    Both tests run via `vitest run`.
