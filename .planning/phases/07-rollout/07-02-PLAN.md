<objective>
Ship the operational documentation surface and the in-app help link that connects the dashboard to the onboarding pages.

Three deliverables: (1) three role-specific markdown one-pagers + a Loom-link index in `docs/onboarding/`, (2) per-country cutover checklist `docs/CUTOVER.md` with one section per active country (Mozambique first, the eleven others as templates), (3) a `?` help link in the bottom of the dashboard sidebar that opens the role-appropriate onboarding page in a new tab.

3. **Common questions** — six to eight role-specific Q&As. For agents: "Why did a lead disappear?" / "What does Follow-ups mean?" / "Why is the date range affecting my counters?" / "What if a lead's number is wrong?" — answers grounded in the actual UI behaviour Phase 3 plan 03-04 shipped. For country admins: reassign mechanics, CSV export, the difference between today's KPI strip and the date-ranged Converted tile. For HQ admins: how status dots are computed (`computeResponseStatus` thresholds: <300s green, ≤480s amber, else red), the leaderboard's drill-in behaviour, why the KPI strip's "Avg Speed to Lead" colour can be green while individual countries are red.

Email addresses pulled from `CREDENTIALS.md`; if the email isn't there, use `william@brainstormprojects.co` + `support@digimountai.com` as the documented v1 contacts (these are the canonical entries to add to `CREDENTIALS.md` if missing — surface in the SUMMARY).

**Per-country checklist (eleven items):**

- [ ] Smoke test from agent seat: log in via invite email, set password, land on `/<cc>/queue`, see (zero or seeded) leads
- [ ] Smoke test from country admin seat: log in, land on `/<cc>`, KPI tiles render zeros, leaderboard shows the agents we provisioned
- [ ] Form-side ingestion flipped to production for this country — Path 1 (direct Paratus webhook to `/api/leads/ingest` with HMAC) or Path 2 (n8n bridge → same endpoint), per the per-country decision

- [ ] 24 h soak: zero `event:'audit_write_failed'` lines, zero 429s on `/api/leads/ingest` from production traffic, zero P1/P2 in Sentry, UptimeRobot uptime ≥ 99.9 %
