<objective>
Add SLA breach detection and email alerts for leads unanswered for ≥5 minutes.

Purpose: PRD/milestones.md Phase 6 requires "country admin gets alerted when an agent doesn't answer in 5 minutes" before pilot launch. Phase 2's ingest path is already shipped (HMAC + dedupe + round-robin + realtime broadcast). What's missing is the alert tail — a cron job that scans for breached leads every minute, emails the country admin via Resend, and dedupes via a `sla_breach_alerted_at` column so duplicate cron deliveries don't double-send.
Output: Migration `00014_sla_alerts.sql` (alerted-at column + `v_sla_breaches` view + `mark_sla_alerted` RPC), Resend wrapper at `packages/supabase/src/lib/email.ts`, React Email template for the breach notification, DAL surface in `packages/supabase/src/dal/sla.ts`, cron route handler at `apps/web/app/api/cron/sla-check/route.ts` registered in `apps/web/vercel.json`, integration test that seeds a breached lead and asserts both the email path and the dedupe behaviour.
</objective>
