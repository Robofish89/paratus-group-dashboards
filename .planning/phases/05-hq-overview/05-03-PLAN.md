**Test suite — 3 specs:**

1. **`golden path: HQ admin lands on / and sees the live overview`**
   - Log in as a fresh `hq_admin` user via the e2e bridge
   - Visit `/`
   - Assert: KPI strip shows 5 (or 4) tiles with non-empty values; "Countries Active" reads `12` (live count from `countries` where `status='active'`)
   - Assert: Country Performance table renders with 12 rows, ordered by total_leads desc
   - Assert: Leads by Service card renders with at least 1 bar
   - Assert: Speed to Lead Trend chart renders an SVG (Recharts) with at least the empty-state or a path element
   - No console errors during page load

2. **`drill-in: clicking a leaderboard row navigates to /[country] and renders country-admin shell`** (RESEARCH.md pitfall 6)
   - Same HQ session
   - Click the country name cell in the first row of the leaderboard (the row with most leads — likely BW or NA depending on seed state)
   - Assert: URL changes to `/<country_code>` (regex match)
   - Assert: country-admin shell renders (e.g. the page heading or a country-admin-specific element appears — e.g. the country-admin sidebar's "Pipeline" nav item, or the country KPI strip's "Total Leads" tile labelled with the country name)
   - Assert: no `403` or redirect to `/login` happened (HQ admin allowed through `(country-admin)/[country]/layout.tsx` per Phase 4 plan 04-03)
   - Document in spec comments: "this test pins the cross-surface contract — any Phase 6 tightening of the country-admin layout role gate MUST keep `hq_admin` in the allow-list, or this test breaks."

3. **`realtime: webhook ingest bumps KPI strip without a page refresh`**
   - HQ session at `/`
   - Capture the initial value of "Total Leads (Group)" from the DOM
   - POST a synthetic lead to `/api/leads/ingest` with valid HMAC + Zod payload (mirror the existing test fixture from `apps/web/tests/realtime.broadcast.test.ts` and from `apps/web/e2e/country-admin-golden-path.spec.ts`'s realtime-bump spec)
   - Wait up to 5s for the tile value to update (Playwright `expect(locator).toHaveText` with a regex matching `initial + 1`, or check that a fresh fetch + render via `router.refresh()` happened by polling for the new value)
   - Assert: the tile value increased by ≥ 1
   - Cleanup: service-role `DELETE FROM leads WHERE id = ...` for the synthetic lead in `afterEach`

**Run:** `npm run e2e --workspace=apps/web -- hq-overview-golden-path` (or whatever the project's playwright command is — verify `apps/web/package.json` for the script name; use the same one Phase 4's spec runs under). **Three consecutive flake-free runs** before declaring green (the project bar set in plan 04-04).
