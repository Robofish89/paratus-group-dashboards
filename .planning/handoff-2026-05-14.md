### Process notes worth keeping

- **Vercel env via stdin gotcha:** `echo "$VAR" | vercel env add ...` adds a trailing `\n` to the stored value. Vercel validates `CRON_SECRET` specifically because it's used as an HTTP `Authorization: Bearer` header — build fails with a clear error. Other vars don't validate but may still cause subtle bugs. Always use `printf '%s' "$VAR" | vercel env add ...`. Saved as `feedback_vercel_env_printf.md` in memory.
- **Vercel login under master account:** `para.group.n8n@gmail.com` logs in as user handle `paragroupn8n-9569` with access to the `paratusgroup` team. The dev's other Vercel account (`robofish89`) only has access to `gerhards-projects-ef4ffd0d` — no overlap with the Paratus team. Re-login required if switching contexts.
- **Afrisp DNS lag:** Paratus's nameserver provider (`ns{1,2}.na.afrisp.net`) has a ~25 min push delay between admin-panel saves and the authoritative server reflecting changes. Don't escalate prematurely.
- **BIND zone-file trailing dot:** MX records on Fanuel's side need explicit FQDN with a trailing dot (`feedback-smtp.eu-west-1.amazonses.com.`) — otherwise BIND treats the value as relative and appends the zone, breaking Resend verification.
