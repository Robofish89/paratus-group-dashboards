<task type="auto">
  <name>Task 2: Migration 00003 — countries + forms reference tables</name>
  <files>packages/supabase/migrations/00003_reference_data.sql</files>
  <action>
    Write and apply migration creating reference tables per PRD/data-model.md.

    1. Create enum: `CREATE TYPE public.country_status AS ENUM ('active', 'coming_soon');`
    2. Create `public.countries` table:
       - `code text PRIMARY KEY` (ISO-3166-1 alpha-2)
       - `name text NOT NULL`
       - `currency text`
       - `timezone text NOT NULL`
       - `status country_status NOT NULL DEFAULT 'active'`
       - `created_at timestamptz NOT NULL DEFAULT now()`
    3. Create `public.forms` table:
       - `slug text PRIMARY KEY`
       - `display_name text NOT NULL`
       - `landing_page_url text`
       - `is_active boolean NOT NULL DEFAULT true`
       - `created_at timestamptz NOT NULL DEFAULT now()`
    4. Enable RLS on both: `ALTER TABLE ... ENABLE ROW LEVEL SECURITY;`
    5. RLS policies — both tables are public read for any authenticated user (reference data is not tenant-scoped):
       ```sql
       CREATE POLICY "countries_read_all" ON public.countries FOR SELECT TO authenticated USING (true);
       CREATE POLICY "forms_read_all" ON public.forms FOR SELECT TO authenticated USING (true);
       ```
       Writes locked: only `service_role` can mutate (no policies for INSERT/UPDATE/DELETE means RLS denies them; service_role bypasses RLS).
    6. Seed 15 countries (12 active + 3 coming_soon) with timezones from PRD/data-model.md:
       Active: AO Angola Africa/Luanda, BW Botswana Africa/Gaborone, CD DRC Africa/Kinshasa, SZ Eswatini Africa/Mbabane, KE Kenya Africa/Nairobi, MZ Mozambique Africa/Maputo, NA Namibia Africa/Windhoek, RW Rwanda Africa/Kigali, ZA "South Africa" Africa/Johannesburg, TZ Tanzania Africa/Dar_es_Salaam, UG Uganda Africa/Kampala, ZM Zambia Africa/Lusaka.
       Coming soon: LS Lesotho Africa/Maseru, MW Malawi Africa/Blantyre, ZW Zimbabwe Africa/Harare.
    7. Seed 10 forms (slug, display_name) per PRD overview:
       general-contact "General Contact", carrier-services "Carrier Services", satellite "Satellite", data-centers "Data Centers", broadband "Broadband", oneweb "OneWeb", starlink "Starlink", essential-access "Essential Access", connect2care "Connect2Care", starlink-for-schools "Starlink for Schools".
    8. Add FK preparedness: no FKs from leads/user_roles yet (those tables come in 02-02). Just `COMMENT ON TABLE` describing intent.

    Apply via `mcp__supabase-paratusgroup__apply_migration`.
