  <files>packages/supabase/migrations/00009_queue_rpcs.sql</files>
  <action>
    Write `00009_queue_rpcs.sql` containing:

    1. **RPC `public.mark_lead_contacted(p_lead_id uuid) returns jsonb`**
       SECURITY DEFINER, search_path locked. Inside the function, verify
       `auth.uid()` matches `leads.assigned_to` AND `leads.country_code` matches
       `auth.jwt() ->> 'country_code'`. If not, raise `'forbidden'`.
       Otherwise UPDATE `leads` SET `status = 'contacted'`,
       `first_contacted_at = COALESCE(first_contacted_at, now())` WHERE id = p_lead_id.
       INSERT INTO `lead_events` (lead_id, actor_id, type, outcome) VALUES
       (p_lead_id, auth.uid(), 'call', 'connected') — actor_id = auth.uid()
       so the agent INSERT policy passes when called via the wrapping route, and
       the trigger fills country_code. Return `jsonb_build_object('lead_id', p_lead_id, 'first_contacted_at', <stamped value>)`.
       Idempotent: if status is already past 'new', the COALESCE on first_contacted_at preserves the original timestamp; the function should still write a fresh `call` event so retries are audit-visible. Do NOT reset status if already qualified/converted/lost — raise `'invalid_status'`.

    2. **RPC `public.complete_call(p_lead_id uuid, p_outcome text, p_notes text default null, p_lost_reason text default null) returns jsonb`**
       SECURITY DEFINER. Same `auth.uid() = assigned_to` guard. `p_outcome` must
       be one of `'qualified' | 'won' | 'lost' | 'no_answer' | 'callback'`
       (validate, raise `'invalid_outcome'` otherwise). Branch on outcome:
         - `qualified` → `leads.status='qualified'`, `qualified_at=now()`
         - `won` → `leads.status='converted'`, `converted_at=now()`
         - `lost` → `leads.status='lost'`, `lost_at=now()`, `lost_reason=p_lost_reason`
         - `no_answer` → leave status='contacted'; just record event
         - `callback` → leave status='contacted'; just record event (the actual
           callback row is written by `schedule_callback`)
       Always INSERT a `call` lead_event with `outcome=p_outcome`, `note=p_notes`,
       `actor_id=auth.uid()`. Return `jsonb_build_object('lead_id', p_lead_id, 'status', <new status>, 'outcome', p_outcome)`.

    3. **RPC `public.schedule_callback(p_lead_id uuid, p_scheduled_for timestamptz, p_notes text default null) returns jsonb`**
       SECURITY DEFINER. Same `auth.uid() = assigned_to` guard. `p_scheduled_for`
       must be `> now()`, raise `'invalid_schedule'` otherwise. INSERT into
       `callbacks` (lead_id, assigned_to, country_code, scheduled_for, status)
       VALUES (p_lead_id, auth.uid(), <derived from leads>, p_scheduled_for, 'pending').
       Also INSERT a `callback_scheduled` lead_event with note=p_notes.
       Return the new callback id.

    4. **VIEW `public.agent_today_stats`** — per-agent counters for the queue header strip:
       ```sql
       CREATE VIEW public.agent_today_stats AS
       SELECT
         ur.user_id AS agent_id,
         ur.country_code,
         count(l.id) FILTER (
           WHERE l.assigned_to = ur.user_id
             AND l.status IN ('new','contacted')
         ) AS to_call_count,
         count(le.id) FILTER (
           WHERE le.actor_id = ur.user_id
             AND le.type = 'call'
             AND le.created_at >= date_trunc('day', now())
         ) AS completed_today,
         count(l.id) FILTER (
           WHERE l.assigned_to = ur.user_id
             AND l.status = 'converted'
             AND l.converted_at >= date_trunc('day', now())
         ) AS converted_today,
         count(cb.id) FILTER (
           WHERE cb.assigned_to = ur.user_id
             AND cb.status = 'pending'
         ) AS callbacks_pending
       FROM public.user_roles ur
       LEFT JOIN public.leads l       ON l.assigned_to = ur.user_id
       LEFT JOIN public.lead_events le ON le.actor_id  = ur.user_id
       LEFT JOIN public.callbacks cb   ON cb.assigned_to = ur.user_id
       WHERE ur.role = 'agent'
       GROUP BY ur.user_id, ur.country_code;
       ALTER VIEW public.agent_today_stats SET (security_invoker = true);
       GRANT SELECT ON public.agent_today_stats TO authenticated;
       ```
       Inherits RLS via `security_invoker = true` — agents see only their own
       row because `leads`, `lead_events`, and `callbacks` policies all gate
       to `assigned_to = auth.uid()`.

    5. **Grants:** `REVOKE ALL ON FUNCTION mark_lead_contacted, complete_call, schedule_callback FROM PUBLIC;`
       `GRANT EXECUTE ON FUNCTION mark_lead_contacted, complete_call, schedule_callback TO authenticated;`
       (Different from `ingest_lead`, which is service-role-only because the
       webhook is HMAC-authed before reaching it. These three are called from
       the agent's authed browser session, so `authenticated` is the right grain.)

    Patterns to mirror: `00007_assignment_function.sql` is the reference for
    `SECURITY DEFINER` + locked search_path + `RAISE EXCEPTION USING ERRCODE`.

    Apply via Supabase MCP (`mcp__supabase-paratus__apply_migration`) or `supabase db push` against the live project `tgswsdfaszvztbpczfve`.
