## Scope (what shipped)

### 1. HQ sidebar stubs replaced with real surfaces

The Phase 5 plan 05-03 sidebar stubs at `/countries`, `/service-mix`, and
`/settings` shipped with placeholder copy promising "Phase 6 will deliver
this." Phase 6 turned into Production Hardening, so the promise was never
met. All three are now real pages:

- **`/countries`** — card grid for all 15 markets (12 active + 3
  coming-soon). Active cards: Total Leads, New Today, Contacted %, Avg
  Response, response-status pill, country-admin + agent head counts,
  drill-in link to `/<slug>`. Coming-soon cards: muted variant with
  explanatory copy.
- **`/service-mix`** — full-width table over `leads_by_service_group`:
  rank, service line, leads, share-of-total, inline distribution bar.
  Diverges from the Overview's bar-chart card by surfacing exact
  figures.
- **`/settings`** — read-only group-admin surface: live `/api/health` DB
  probe (status pill, ms, deployed commit SHA, region), group footprint
  (active/coming-soon counts, head counts), SLA threshold reference,
  per-country audit log entry points, "future iteration" note for group
  admin controls.

### 2. Country-admin scoping leak fixed

`apps/web/app/(country-admin)/[country]/leads/page.tsx` had every filter
except `country_code`. Country admins were RLS-pinned correctly, but HQ
admins (allow-listed by `requireRole`) saw rows from every country
regardless of which slug was in the URL. Verified in production: the only
seeded leads were 9 MZ rows, which appeared on `/cd/leads`, `/bw/leads`,
etc. when an HQ admin opened them.

- Lead-list query now applies `eq('country_code', countryCode)`
  unconditionally.
- `/api/country-admin/export-leads` now accepts `?country=<code>` and
  applies the eq filter; the leads page passes it; filename uses the
  explicit filter when present (`leads-CD-…csv` not `leads-all-…csv`).

### 3. Empty-state polish

- **`computeResponseStatus`** gains a `'none'` variant (neutral grey)
  gated behind a `{ hasData }` option. Empty / freshly-seeded countries no
  longer alarm red; distinguishes "no signal yet" from "leads exist but
  none contacted." Existing call sites opt in (CountryCard,
  country-leaderboard, HQ KPI strip avg-speed accent). New table-driven
  tests for the empty-state branch.
- **Country-admin Overview** shows an "Awaiting first lead for [Country]"
  slate banner above the cards when `total_leads === 0`. Only renders on
  zero-lead countries; hidden when real data exists.

### 4. Phase 6 audit log surfaced on dashboard layer

- **HQ Overview** gains a "Recent group activity" panel: last 10 audit
  events across the group with action chip, country link, actor + role,
  timestamp linking to the per-country audit page.
- **Country-admin Overview** gains a compact "Recent activity" teaser:
  last 5 events for that country with "View all →" link to the canonical
  `/<country>/audit` page.
- DAL: `getRecentGroupActivity(limit, countryCode?)` — single helper
  serves both surfaces; RLS gates row visibility.

### 5. Country-admin Settings page

`/[country]/settings` was a dead nav item that 404'd. Now a real
read-only page mirroring the HQ Settings pattern: country profile (ISO,
timezone, currency, status pill), group SLA thresholds, country admin
roster, agent roster, audit log link, provisioning explanation.
