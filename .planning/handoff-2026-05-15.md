### The HMAC catch (do not forget this on resume)

`apps/web/app/api/leads/ingest/route.ts` is **strictly HMAC-only** (lines 86–95): requires header `x-paratus-signature` = hex HMAC-SHA256 of the raw JSON body, signed with `PARATUS_INGEST_SECRET`. No bearer/IP/query fallback.

Elementor Pro's **native Webhook action cannot do this** — no HMAC, no custom headers via UI, no payload reshaping. The solve (keeps Phase 2 security model fully intact):

> A WordPress **code snippet** using the `elementor_pro/forms/webhooks/request_args` filter — intercepts each form's outgoing webhook, reshapes the payload into `ingestSchema`, computes the HMAC, injects `x-paratus-signature`. ~30 lines, one snippet, all forms. Requires ability to add PHP → **WordPress Administrator**, or Editor + admin on a code-snippets plugin (WPCode / Code Snippets).
